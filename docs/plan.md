# W4Agent 毕业论文写作计划

## 一、论文基本定位

**拟定题目：** 基于智能体技术的 Web 无障碍检测系统设计与实现

**研究对象：** W4Agent——基于多智能体协同、自适应网页遍历、规则与大模型融合检测的 Web 无障碍检测系统。

**论文类型：** 软件工程实践型 + 方法改进型毕业论文。

**目标字数：** 正文不少于 2 万字，建议最终控制在 2.5 万～3 万字。正文不含摘要、目录、参考文献、致谢和附录。

**核心论述主线：**

1. Web 无障碍检测具有现实意义，传统自动化检测工具在动态页面、语义理解、交互路径覆盖等方面存在不足。
2. W4Agent 引入 LLM Agent、多智能体编排、自适应遍历、规则-语义-视觉融合检测机制，提高检测流程的智能化和自动化程度。
3. 系统采用 FastAPI + PostgreSQL/Redis + Playwright + LangGraph/LangChain + React + Ant Design 的前后端分离架构，实现任务创建、自动遍历、无障碍检测、问题标注、报告生成和可视化工作台。
4. 通过功能测试、模块测试和对比实验验证系统的可用性、覆盖能力和检测效果。

## 四、详细论文目录与写作要点

### 摘要

**建议字数：** 中文摘要 500～800 字；英文摘要 300～500 词。

**写作要点：**

- 说明 Web 无障碍检测的现实背景和标准依据。
- 概括传统检测工具在动态页面遍历、语义理解和视觉一致性检测方面的不足。
- 提出 W4Agent 系统的核心方案：多智能体协同、自适应遍历、规则-语义-视觉融合检测、人机协同工作台。
- 总结系统实现和测试结果。

**关键词：** Web 无障碍；智能体；大语言模型；自适应遍历；WCAG；多模态检测

### 第 1 章 绪论

**建议字数：** 3000～3500 字。

#### 1.1 研究背景

- 介绍互联网服务在政务、教育、医疗、购物等场景中的基础性作用。
- 说明老年人、视障人士、听障人士、肢体障碍人士等群体对无障碍访问的需求。
- 引入 WCAG 2.1、GB/T 37668-2019、《无障碍环境建设法》等标准和政策背景。
- 说明 Web 无障碍检测在网站建设、合规评估和用户体验优化中的价值。

#### 1.2 国内外研究现状

- 介绍传统规则检测工具，如 axe-core、WAVE、Lighthouse、Pa11y 等。
- 分析规则检测工具的优点：速度快、确定性强、容易工程化。
- 分析不足：难以理解语义、难以处理动态页面、容易遗漏交互路径、视觉语义一致性判断不足。
- 介绍 LLM Agent、浏览器自动化、多模态模型在软件测试和网页理解中的应用趋势。

#### 1.3 研究问题与挑战

- 问题一：复杂网站中如何提高检测页面和交互路径覆盖率。
- 问题二：动态弹窗、Cookie 横幅、广告遮挡、Alert 等突发事件如何自动处理。
- 问题三：如何在规则检测之外发现语义不清、标签不准确、视觉与文本不一致等问题。
- 问题四：如何将检测过程、问题列表、截图标注和报告生成集成为完整工作流。

#### 1.4 研究内容与主要贡献

- 设计基于 LangGraph 的多智能体协同检测流程。
- 设计基于 Playwright 的自适应智能遍历器。
- 实现规则检测、LLM 语义分析和多模态视觉分析结合的检测引擎。
- 构建前后端分离的人机协同工作台。
- 实现截图标注、报告管理和任务实时监控。

### 第 3 章 系统需求分析

**建议字数：** 2500～3000 字。

#### 3.1 系统目标

- 支持创建 Web 无障碍检测任务。
- 支持自动遍历目标网站页面。
- 支持基于规则和 AI 的无障碍问题检测。
- 支持实时查看任务进度、Agent 日志和检测结果。
- 支持报告生成、截图标注、人工复核与结果管理。

#### 3.2 用户角色分析

- 系统管理员：管理系统配置、模型参数、检测策略。
- 测试人员：创建任务、查看检测过程、复核检测结果。
- 项目负责人：查看报告、评估网站无障碍质量。

#### 3.3 功能需求分析

- 项目管理：创建、查询、更新项目。
- 任务管理：创建检测任务、配置目标 URL、检测深度、规则等级。
- 智能遍历：自动访问页面、分析链接、去重、处理弹窗。
- 无障碍检测：执行 WCAG 规则检测和 AI 分析。
- 实时监控：通过 WebSocket 推送任务状态、进度和日志。
- 报告管理：统计问题数量、严重程度、页面分布并生成报告。
- 人工标注：查看标注截图，对问题进行确认、忽略或修改。

#### 3.4 非功能需求分析

- 可扩展性：规则可注册扩展，Agent 可扩展，LLM Provider 可替换。
- 可靠性：异常页面、弹窗、网络错误不应导致任务整体失败。
- 易用性：前端工作台需要直观展示任务和问题。
- 性能：浏览器池复用，减少重复页面遍历，控制 LLM 调用成本。
- 可维护性：前后端分层清晰，模块职责明确。

#### 3.5 业务流程分析

- 用户创建检测任务。
- 后端启动 Agent Engine。
- Master Agent 规划任务阶段。
- Crawler Agent 调用自适应遍历器收集页面。
- Detector Agent 调用检测引擎分析问题。
- Reporter Agent 汇总检测结果。
- 前端实时展示检测进度和报告。

#### 4.1 总体架构设计

#### 4.2 后端服务架构设计

- FastAPI 应用入口、生命周期管理、CORS、异常处理、静态截图服务。
- API 路由按 projects、tasks、pages、issues、reports、annotations 划分。

#### 4.3 智能体引擎总体设计

- Agent Engine 作为任务入口。
- Orchestrator 负责状态图运行。
- Master Agent 负责全局调度。
- Crawler Agent、Detector Agent、Reporter Agent 分别处理探索、检测和报告。
- 短期记忆保存单次任务上下文，长期记忆保存跨任务经验。

- 浏览器池负责页面执行环境。

- Deduplicator 进行精确哈希和结构签名去重。
- EventResponder 处理 Cookie 横幅、弹窗、广告和 Alert。
- StateGraph 维护页面访问状态和跳转关系。

#### 4.5 检测子系统设计

- DetectionEngine 统一协调规则检测和 AI 检测。
- Rule Registry 负责规则注册和批量执行。
- WCAG Rules 覆盖确定性规则。
- Detector Agent 对 A11y Tree、HTML 片段和规则结果进行语义分析。
- Visual Analyzer 结合截图和元素边界框进行视觉一致性判断。

#### 4.6 前端工作台设计

- 页面组成：Dashboard、ProjectList、TaskCreate、TaskDetail、ReportView、Settings。

- 标注组件：AnnotatedScreenshot。
- 说明任务状态、问题统计、报告查看、截图标注的交互方式。

#### 4.7 数据库设计

- 项目表：保存被测项目基本信息。
- 任务表：保存检测任务配置、状态、进度。
- 页面表：保存已遍历页面 URL、标题、截图、检测状态。
- 问题表：保存无障碍问题类型、严重程度、规则编号、位置。
- 报告表：保存汇总统计和报告内容。

- WebSocket：用于任务进度、Agent 日志、实时检测结果推送。

| TC05 | 表单检测 | 缺少 label 的输入框 | 输出表单标签问题 |
| TC06 | 报告生成 | 完成的检测任务 | 生成问题统计和修复建议 |
| TC07 | 截图标注 | 有定位信息的问题 | 前端显示标注框和严重程度颜色 |
| TC08 | WebSocket 监控 | 运行中的任务 | 前端实时更新进度和日志 |

#### 6.3 模块测试

- Agent 编排测试：验证不同阶段状态转移是否正确。
- 遍历器测试：验证 URL 评分、去重和事件响应。
- 检测引擎测试：验证规则注册、规则执行和 AI 合并结果。
- 前端测试：验证路由、页面展示和报告交互。

#### 6.4 对比实验设计

对比对象建议：axe-core、WAVE、Lighthouse 或 Pa11y。

实验网站建议分为：

- 静态内容型网站。
- 表单交互型网站。
- 单页应用 SPA。
- 电商/列表模板型网站。
- 含弹窗或 Cookie 横幅的网站。

对比指标：

- 页面覆盖率：实际检测页面数 / 可访问页面数。
- 问题发现数量：不同严重程度问题数量。
- 准确率：人工复核后真实问题数 / 检测问题总数。
- 召回率：检测出的真实问题数 / 人工标注真实问题总数。
- 误报率：误报问题数 / 检测问题总数。
- 平均检测耗时：任务总耗时 / 页面数。
- LLM 调用次数与成本：每个任务或每页平均调用成本。

### 第 7 章 总结与展望

**建议字数：** 1500～2000 字。

#### 7.1 工作总结

- 完成了一套基于智能体技术的 Web 无障碍检测系统。
- 实现了从任务创建、智能遍历、检测分析、截图标注到报告生成的闭环流程。
- 实现了多智能体编排、自适应遍历、规则和 AI 融合检测等核心功能。

#### 7.2 创新点总结

- 多智能体协同的检测任务编排机制。
- 面向无障碍检测的自适应网页遍历策略。
- 规则-语义-视觉融合的多层检测方法。
- 结合实时监控、可视化标注和人工复核的人机协同工作台。

#### 7.3 不足与改进方向

- LLM 调用成本和响应时间仍需优化。
- AI 语义检测存在一定误报，需要引入更强的人工复核和置信度机制。
- 对复杂登录态、多步骤业务流程的支持仍可增强。
- 可进一步丰富规则库，覆盖更多 WCAG 成功准则。
- 可增加批量检测、团队协作、CI/CD 集成能力。
