中期报告内容                                                                                                                                                                
                                                                                                                                                                              
  一、已完成的内容                                                                                                                                                            
                                                                                                                                                                              
  1. 系统整体架构设计与搭建                                                                                                                                                   
  - 完成了前后端分离的整体架构设计：后端 Python + FastAPI，前端 React + TypeScript + Ant Design                                                                               
  - 完成 Docker Compose 部署方案，包含 PostgreSQL(pgvector)、Redis、后端、前端四个服务
  - 完成数据库设计，定义了 7 个核心模型（Project、Task、Page、Issue、Report、Annotation、AgentMemory）
  - 完成 RESTful API 层（6组路由：项目/任务/报告/问题/页面/标注）和 WebSocket 实时通信机制

  2. 智能体引擎（核心模块一）                                                                                                                                                 
  - 基于 LangGraph 实现了完整的多智能体编排状态机（Orchestrator），包含 Plan → Explore → Test → Report 四个节点的循环决策流程                                                 
  - 实现了四个专用 Agent：MasterAgent（任务规划决策）、CrawlerAgent（爬取策略建议）、DetectorAgent（AI无障碍检测 + 多模态视觉分析）、ReporterAgent（报告生成）                
  - 完成 LLM Provider 工厂，支持 OpenAI、Anthropic Claude、本地 Ollama 三种模型后端                                                                                           
  - 实现了短期记忆（bounded deque，100条上限）和长期记忆（PostgreSQL持久化存储，支持按领域召回历史经验）                                                                      
                                                                                                                                                                              
  3. 自适应遍历器（核心模块二）                                                                                                                                               
  - 实现了 AdaptiveCrawler，基于 Playwright 进行浏览器自动化导航                                                                                                              
  - 实现启发式 URL 优先级排序策略（HeuristicStrategy），根据页面类型、深度、父页面特征等多因素综合打分
  - 实现基于内容哈希和 DOM 结构签名的语义去重（SemanticDeduplicator），可识别模板化页面并避免冗余检测
  - 实现应用状态图（ApplicationStateGraph），以有向图形式跟踪页面探索状态
  - 实现页面分析器（PageAnalyzer），提取完整的无障碍树（A11y Tree）、DOM结构信息和元素边界框

  4. 突发事件响应器（核心模块三）
  - 实现了 EventResponder，可检测和自动处理 Cookie 横幅、弹窗/模态框、广告覆盖层、Alert 对话框四类突发事件
  - 支持中英文双语按钮模式匹配（Accept/接受、Close/关闭等），具备 Escape 键降级策略
  - 事件检测结果通过 WebSocket 实时通知前端

  5. 人机协同工作台（核心模块四）
  - 完成 Dashboard 首页（任务统计、最近任务列表）
  - 完成项目管理页面（CRUD操作）
  - 完成任务创建页面（目标URL、WCAG级别、检测配置等参数设置）
  - 完成任务详情页面（实时进度监控、问题列表、Agent推理日志时间线、页面截图与标注四个Tab页）
  - 完成报告查看页面（分数仪表盘、严重度分布、问题详情、导出HTML/PDF/JSON）
  - 实现带标注的截图查看组件（AnnotatedScreenshot），支持边界框高亮和严重度色彩编码
  - 实现 WebSocket 客户端，支持自动重连和指数退避

  6. 无障碍检测引擎
  - 实现了规则检测引擎框架（DetectionEngine + RuleRegistry），支持按 WCAG 级别过滤
  - 实现了 7 条 WCAG Level A 规则：图片替代文本（1.1.1）、替代文本质量（1.1.1）、表单标签（1.3.1）、标题层级（1.3.1）、页面语言（3.1.1）、地标区域（1.3.1）、链接文本（2.4.4）
  - 实现了 AI 辅助检测（DetectorAgent 基于 LLM 分析 A11y 树和 HTML）
  - 实现了多模态视觉检测（将截图和元素边界框发送至视觉 LLM 进行分析）
  - 实现了可视化标注系统（VisualAnnotator），在截图上绘制问题标注框、编号标签和图例

  7. 端到端数据流打通
  - 前端创建任务 → 后端 API 接收 → 后台启动 Agent Pipeline → 多轮 Plan/Explore/Test 循环 → 生成报告，整个流程已完全打通
  - 实时进度通过 WebSocket 从后端推送至前端展示

  三、存在的问题和解决办法

  问题1：WCAG 规则覆盖率有限
  - 现状：目前仅实现了 7 条 Level A 规则，WCAG 2.1 共有 78 条成功标准，规则覆盖率较低
  - 解决办法：优先补充 AA 级别的高影响力规则（颜色对比度、键盘可访问性、焦点可见等）；同时依赖 AI 检测（DetectorAgent）来弥补规则覆盖的不足，利用 LLM
  的语义理解能力发现规则难以覆盖的问题

  问题2：缺少移动应用检测能力
  - 现状：任务书中提到"移动应用"检测，但当前系统仅支持 Web 端检测，未集成 Appium 等移动测试框架
  - 解决办法：考虑到时间限制，在论文中将系统定位为"以 Web 为主的无障碍检测系统"，通过移动端视口模拟（viewport 配置）和响应式页面检测部分覆盖移动 Web
  场景；后续可作为扩展工作讨论 Appium 集成方案

  问题3：缺少量化实验验证
  - 现状：系统已开发完成但缺少系统性的测试实验，未产出遍历覆盖率提升、事件处理成功率、检测准确率等关键指标数据
  - 解决办法：选择 2-3 个不同类型的代表性网站（政府网站、电商网站、新闻网站等），设计对比实验（与传统 DFS 遍历对比覆盖率、与人工专家结果对比
  Precision/Recall），收集量化数据支撑论文结论

  问题4：人机协同闭环功能未完善
  - 现状：后端的标注 API（确认/误报标记/重分类/评论）已实现，但前端标注审核页面仍为占位符
  - 解决办法：基于已有的后端 API 和前端组件库，快速开发标注审核页面；核心功能包括：问题逐条审核、一键标记误报、添加人工注释

  问题5：LLM 调用成本与延迟
  - 现状：每次检测任务需要多轮 LLM 调用（MasterAgent 决策 + DetectorAgent 检测 + ReporterAgent 报告），在页面较多时成本和延迟较高
  - 解决办法：引入分级检测策略——简单页面仅用规则检测，复杂页面才启用 AI 检测；同时利用本地 Ollama 模型降低 API 调用成本
